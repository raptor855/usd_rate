### Разбиение на подзадачи 
## Аналитика

 Уточнить требования к:

- источнику курса (API, валюта, формат)
  - формату JSON-ответа
  - способу хранения истории запросов
  - трактовке «пауза не менее 10 секунд»
- Выбрать внешнее API для курса USD → RUB.

Архитектура

- Создать Django-проект и приложение.
- Спроектировать модель для хранения запросов курса.
- Реализовать сервис получения курса.
- Реализовать механизм ограничения запросов (≥ 10 секунд).
- Реализовать HTTP endpoint /get-current-usd/.
- Подготовить сериализацию ответа в JSON.
- Добавить минимальные настройки и документацию.

### Вопросы
## Продукт-менеджеру

- Источник курса критичен или можно использовать любой публичный API?
- Нужно ли учитывать кэширование (например, Redis)?
- Ограничение 10 секунд — глобальное или на клиента?
- Что важнее: скорость ответа или строгое соблюдение интервала?
- Требуется ли авторизация?

## Tech Lead

- Допустимо ли использовать SQLite для тестового задания?
- Нужно ли покрывать код тестами?
- Как предпочтительнее реализовать rate-limit:
  - через БД
  - через cache (Redis)
- Нужен ли Docker?
- Требуется ли логирование ошибок внешнего API?
